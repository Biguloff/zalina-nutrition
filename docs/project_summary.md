## Дата: 03.04.2026

## 1. Обзор проекта

**Цель:** Построить полуавтоматизированную систему ведения клиентов для нутрициолога-натуропата Залины Бигуловой. Превратить ручной процесс (1200 сообщений/мес на клиента, 85% шаблонных) в систему с ИИ-ботом, CRM-дашбордом и структурированным хранением данных.

**Что было сделано в этой сессии:**
1. Глубокий анализ голосовой транскрипции бизнес-процессов (Схема.md)
2. Анализ реальных чатов Telegram (3187 сообщений, 61 день, 12 клиентов)
3. Интерактивная mindmap процессов
4. Линейная карта бизнес-процессов v1.0 (16 шагов, 7 фаз)
5. Линейная карта v2.0 с тремя колонками: Клиент / ИИ-бот / Нутрициолог
6. Архитектура системы (CRM, БД, папки, карточка клиента)
7. Полнофункциональный веб-дашборд с реальными данными 2 клиентов
8. Деплой всего на сервер

## 5. Технологический стек ИИ

### По этапам процесса

| Этап | Инструмент | Стоимость | Что делает |
|------|-----------|-----------|-----------|
| **Распознавание речи** (Zoom) | OpenAI Whisper API | $0.006/мин (~$0.72 за 2ч) | Zoom запись → ffmpeg chunks → Whisper → JSON с таймкодами |
| **Анализ фото еды** | GPT-4o Vision API | $0.002/фото | Фото → распознавание продуктов → оценка БЖУ → «правило тарелки» |
| **Структурирование заметок** | Claude Sonnet 4 API | $0.10/консультация | Транскрипт → structured JSON (резюме, вопросы, действия, FAQ) |
| **Предварительный разбор анализов** | Claude / GPT-4o | ~$0.05/разбор | OCR фото анализов → таблица маркеров → маркировка ↑↓ → черновик |
| **Коллаж До/После** | html2canvas (браузер) | Бесплатно | HTML/CSS → canvas → PNG/JPEG экспорт |
| **Графики** | ApexCharts (MIT) | Бесплатно | Вес, замеры, динамика — time-series с zoom |
| **Хостинг** | GitHub Pages / VPS | Бесплатно | Статический SPA + кастомный домен + HTTPS |
| **База данных** | Supabase (free tier) | Бесплатно (500MB) | PostgreSQL + Auth + Storage + REST API |
| **Автоматизация** | n8n / GitHub Actions | Бесплатно | Webhook, cron, обработка записей |
| **Telegram-бот** | python-telegram-bot | Бесплатно | Рассылки, опросы, сбор данных |

**Итого: ~$5-10/мес** на все ИИ-сервисы для практики с 5-10 активными клиентами.

## 9. Схема базы данных (8 таблиц)

### clients
`id` UUID PK | `last_name` text | `first_name` text | `phone` text | `telegram` text | `status` enum(потенциальный/активный/на_паузе/завершён) | `service_type` enum | `program_stage` text | `cloud_folder_url` url | `created_at` datetime

### measurements
`id` UUID PK | `client_id` → clients | `date` date | `weight_kg` num | `waist_cm` num | `hip_cm` num | `belly_cm` num | `bmi` calc

### lab_results
`id` UUID PK | `client_id` → clients | `test_date` date | `test_type` enum(ОАК/Биохимия/Гормоны/Витамины) | `file_path` url | `status` enum(новый/разобран)

### lab_markers
`id` UUID PK | `lab_result_id` → lab_results | `marker_name` text | `value` num | `unit` text | `ref_low` num | `ref_high` num | `status` enum(↓/✓/↑/⚠)

### supplement_protocols + supplement_items
`protocol_id` → clients | `version` int | `start_date` date | `end_date` date | Items: `supplement_name` | `dosage` | `timing` | `purchase_url`

### appointments
`client_id` → clients | `date` datetime | `type` enum(первичная/еженедельная/разбор/итоговая) | `duration_min` int | `zoom_link` url | `notes` text

### food_diary
`client_id` → clients | `date` date | `meal_type` enum(завтрак/обед/ужин/перекус) | `photo_url` url | `ai_assessment` text | `feedback` text

### daily_tracking
`client_id` → clients | `date` date | `weight_kg` num | `water_liters` num | `no_sugar` bool | `exercise` text | `psychotechnique` int | `day_type` enum(этапный/перерыв) | `mood` 1-10

### payments
